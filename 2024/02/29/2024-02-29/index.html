blog 重启，我回来啦

转眼间已经断更了快两年了，看了一眼上一次更新还是在刚进“厂子”里的时候。现在身份转变了，作为独立开发者我决定继续稳定更新 blog 。记录我的个人项目开发以及一些个人的思考和启发。

重启后的碎碎念

简单记录一下这几天做的事

blog 重启计划

其实过年前就有这个计划，那个时候还在开发 “洋葱拦截器” 项目以及适应新的生活节奏。近几天在筹备我计划的 B 站第一个视频合集 《手把手开发组件库》的时候决定继续更新 blog 以及创建一个个人 wiki 开对过程做一个记录以及文档归档。

刚开始落地就遇到了一个小“插曲”，由于之前的 hexo 项目在老笔记本上面，然而老笔记本已经开不了机了。。。 这就导致我 从21 年之后的博客源码找不回来了。于是我也只能把遗漏的部分硬生生搬运到了github 上托管的 hexo 项目中了。

为了这次“blog 重启计划”我还做了以下几件事

  • 引入了评论区（为了方便实现评论区还换了主题）

  • 引入了“请我喝杯咖啡”的按钮

  • 做了双平台部署加push github 仓库的流程（避免再次丢失信息）

个人 wiki

为了更好地管理个人的一些 笔记以及项目文档，我还创建了一个 个人文档库 “Eric’s Wiki”

运用 VitePress 搭建，后续主要计划的栏目有：源码地图、刷题笔记、读书笔记、个人项目等

聊聊短期的安排

近期主要的工作还是先实现 B 站“组件库”实战视频的准备工作，目前已经形成了一套适配该项目的 DevOps 工作流

  • 计划：Obsidian + ChatGPT 进行项目规划、模块设计以及文档记录

  • 开发：TDD（vitest 单测先行） -> VsCode 作为开发工具 + AI 辅助生成 测试用例以及部分代码

  • 构建： Vite(本地) + Github Actions (线上)

  • 测试： Vitest + Cypress （单元 + e2e）

  • 部署： Github Actions 自动部署 -> Github Pages 托管

  • 制品管理： npm 仓库

  • 文档管理： VitePress + Github Wiki + READEME.md

  • 缺陷管理：Github Issues + Github Actions

眼下的重点就是先把组件库的主要组件开发完毕，形成较为美观的组件文档以及合理的章节设计。

后续我也会时常在这里同步进度

继续加油吧～
